# Pi-Chat Public Site

## What this repo is

This is the **public website and documentation site** for [pi-chat.org](https://pi-chat.org).

Pi-Chat is an open-source, self-hosted Slack alternative built by the Pascack Pioneers team. The actual chat workspace (the application users sign into) lives in a separate repository at `https://github.com/alextheradu/pi-chat`. This repo is only the public-facing marketing site and docs companion served at pi-chat.org.

## Tech stack

- Next.js 16 App Router, statically exported via `output: 'export'`
- Markdown docs in `content/docs/` rendered server-side with `marked`
- Framer Motion for page transitions
- No runtime server, no database, no auth. Purely static output served from `out/`
- Served in production with `python3 -m http.server` from the `out/` directory

## Project structure

```
app/            Next.js routes (page.tsx, layout.tsx, docs/)
components/     React components (MarketingLandingPage, PublicNavbar, DocsShell)
content/docs/   Markdown source files for documentation pages
lib/            Site config, docs registry, markdown renderer
out/            Static build output (generated by npm run build)
```

## Project Structure and Module Organization

App routes live in `app/`, including the homepage in `app/page.tsx`, the docs index in `app/docs/page.tsx`, and per-doc routes in `app/docs/[slug]/page.tsx`. Reusable UI belongs in `components/`. Shared helpers such as docs loading and hard-coded site metadata live in `lib/`. Markdown source for public docs is stored in `content/docs/`. When adding a new document, keep the filename, slug, and `lib/docs.ts` entry aligned.

## Dev commands

```bash
npm install      # install dependencies
npm run dev      # Next.js dev server on port 3002
npm run build    # static export to out/
npm run start    # serve out/ with python3 on port 3002
npm run typecheck
```

## Writing style for docs

- No em-dashes and no en-dashes as punctuation. Use commas, colons, or rewrite the sentence.
- Short sentences. Active voice. Plain English.
- Describe Pi-Chat (the chat app) as a separate project from this site.

## Coding Style and Naming Conventions

Follow the existing TypeScript and React style: 2-space indentation, single quotes, and no semicolons. Name React components and exported types in PascalCase, helpers and variables in camelCase, and docs filenames in lowercase kebab-case such as `getting-started.md`. Prefer the `@/` import alias for internal modules. Keep components focused; move data loading and parsing into `lib/` when logic is reused.

## Testing Guidelines

There is no automated test suite in this checkout. Treat `npm run typecheck` and `npm run build` as the minimum verification set. For content or UI changes, manually check `/`, `/docs`, and the affected `/docs/[slug]` page. Confirm outbound links still point at the hard-coded repository and domain values in `lib/site-config.ts`.

## Configuration and Content Tips

Branding and repository settings are intentionally hard-coded in `lib/site-config.ts` so the site can be exported and hosted as plain static files. This repo is the public companion app, not the private chat workspace. Avoid adding authenticated runtime behavior here unless the architecture is intentionally changing.
