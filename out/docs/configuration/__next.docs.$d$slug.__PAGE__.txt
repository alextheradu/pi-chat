3:Te5c,<h1 id="configuration">Configuration</h1><p>Pi-Chat is configured entirely through environment variables. Copy <code>.env.example</code> to <code>.env</code> and set the values described below before running the app.</p>
<h2 id="database">Database</h2><table>
<thead>
<tr>
<th>Variable</th>
<th>Description</th>
</tr>
</thead>

</tr>
</tbody></table>
<h2 id="authentication">Authentication</h2><table>
<thead>
<tr>
<th>Variable</th>
<th>Description</th>
</tr>
</thead>
<tbody><tr>
<td><code>NEXTAUTH_URL</code></td>
<td>Full public URL of your deployment</td>
</tr>
<tr>
<td><code>NEXTAUTH_SECRET</code></td>
<td>Random secret used to sign session tokens</td>
</tr>
<tr>
<td><code>GOOGLE_CLIENT_ID</code></td>
<td>Google OAuth client ID</td>
</tr>
<tr>
<td><code>GOOGLE_CLIENT_SECRET</code></td>
<td>Google OAuth client secret</td>
</tr>
</tbody></table>
<p>Generate a strong <code>NEXTAUTH_SECRET</code> with:</p>
<pre><code class="language-bash">openssl rand -base64 32
</code></pre>
<h2 id="access-policy">Access policy</h2><table>
<thead>
<tr>
<th>Variable</th>
<th>Description</th>
</tr>
</thead>
<tbody><tr>
<td><code>ALLOWED_DOMAIN</code></td>
<td>Google accounts in this domain can sign in without an invite</td>
</tr>
<tr>
<td><code>ADMIN_EMAIL</code></td>
<td>Email address that receives admin role on first sign-in</td>
</tr>
</tbody></table>
<p>Only accounts whose email ends with <code>@ALLOWED_DOMAIN</code> are allowed to sign in by default. All other users need an explicit invite from an admin.</p>

<tr>

<td>Storage endpoint port (default <code>9000</code>)</td>
</tr>
<tr>
<td><code>MINIO_ACCESS_KEY</code></td>
<td>Access key</td>
</tr>
<tr>
<td><code>MINIO_SECRET_KEY</code></td>
<td>Secret key</td>
</tr>
<tr>
<td><code>MINIO_BUCKET</code></td>

<h2 id="google-oauth-setup">Google OAuth setup</h2><ol>
<li>Go to the <a href="https://console.cloud.google.com/">Google Cloud Console</a>.</li>
<li>Create a project and enable the Google+ API.</li>
<li>Create OAuth 2.0 credentials with type Web Application.</li>
<li>Add <code>http://localhost:3001/api/auth/callback/google</code> as an authorized redirect URI for local development.</li>
<li>Add your production URL with the same callback path for production.</li>
<li>Copy the client ID and secret into <code>.env</code>.</li>
</ol>
