11:Tc36,<h1 id="self-hosting">Self-Hosting</h1><p>Pi-Chat is designed to run on infrastructure you control. The recommended production path uses Docker Compose with the included configuration.</p>
<h2 id="what-you-need">What you need</h2><ul>
<li>A server running Docker and Docker Compose</li>
<li>A domain name with HTTPS (required for Google OAuth)</li>
<li>A Google Cloud project with OAuth credentials</li>
<li>Outbound internet access for Google sign-in callbacks</li>
</ul>
<h2 id="clone-and-configure">Clone and configure</h2><pre><code class="language-bash">git clone https://github.com/alextheradu/pi-chat.git
cd pi-chat

cp .env.example .env
</code></pre>
<p>Edit <code>.env</code> with your production values. Key variables to set:</p>
<pre><code class="language-env">DATABASE_URL=&quot;postgresql://pichat:yourpassword@db:5432/pichat&quot;
NEXTAUTH_URL=&quot;https://chat.yourdomain.com&quot;
NEXTAUTH_SECRET=&quot;$(openssl rand -base64 32)&quot;
GOOGLE_CLIENT_ID=&quot;your-client-id&quot;
GOOGLE_CLIENT_SECRET=&quot;your-client-secret&quot;
ALLOWED_DOMAIN=&quot;yourdomain.com&quot;
ADMIN_EMAIL=&quot;admin@yourdomain.com&quot;
MINIO_ENDPOINT=&quot;minio&quot;
MINIO_PORT=&quot;9000&quot;
MINIO_ACCESS_KEY=&quot;your-access-key&quot;
MINIO_SECRET_KEY=&quot;your-secret-key&quot;
</code></pre>
<h2 id="start-the-stack">Start the stack</h2><pre><code class="language-bash">docker compose up --build -d
</code></pre>
<p>This starts three containers:</p>
<ul>
<li><code>app</code>: the Next.js server</li>
<li><code>db</code>: PostgreSQL</li>
<li><code>minio</code>: object storage</li>
</ul>
<p>The entrypoint automatically applies migrations, creates storage buckets, and runs the bootstrap seed before starting the server.</p>
<h2 id="check-health">Check health</h2><pre><code class="language-bash">curl https://chat.yourdomain.com/api/health
</code></pre>
<p>Expected response:</p>
<pre><code class="language-json">{ &quot;db&quot;: &quot;ok&quot;, &quot;minio&quot;: &quot;ok&quot; }
</code></pre>
<h2 id="view-logs">View logs</h2><pre><code class="language-bash">docker compose logs -f app
</code></pre>
<h2 id="reverse-proxy">Reverse proxy</h2><p>Put Nginx, Caddy, or a cloud load balancer in front of the app container. The Next.js server listens on port <code>3001</code> by default. Configure your proxy to terminate TLS and forward to that port.</p>
<p>Caddy example:</p>
<pre><code>chat.yourdomain.com {
  reverse_proxy localhost:3001
}
</code></pre>
<h2 id="updating">Updating</h2><pre><code class="language-bash">git pull
docker compose up --build -d
</code></pre>
<p>Migrations run automatically on each container restart.</p>
<h2 id="backup">Backup</h2><p>Back up PostgreSQL with:</p>
<pre><code class="language-bash">docker compose exec db pg_dump -U pichat pichat &gt; backup.sql
</code></pre>
<p>Back up MinIO by copying the bucket data, or configure MinIO with a remote backend that handles its own redundancy.</p>
<h2 id="environment-variable-reference">Environment variable reference</h2><p>See the <a href="/docs/configuration/">Configuration</a> doc for the full list of environment variables.</p>
