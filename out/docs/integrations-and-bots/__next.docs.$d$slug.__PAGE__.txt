3:Tac7,<h1 id="integrations-and-bots">Integrations and Bots</h1><p>The current branch now includes a first integrations slice: incoming webhooks.</p>
<h2 id="what-exists-today">What exists today</h2><p>The current branch supports:</p>
<ul>
<li>webhook creation by authorized operator accounts</li>
<li>a dedicated bot user per webhook</li>
<li>channel-targeted message delivery from external applications</li>
<li>webhook revocation</li>
</ul>
<h2 id="management-page">Management page</h2><p>Authorized users can manage webhooks at:</p>
<ul>
<li><code>/integrations</code></li>
</ul>
<p>Current permission rule:</p>
<ul>
<li><code>ADMIN</code> and <code>MODERATOR</code> can manage incoming webhooks</li>
</ul>
<h2 id="management-api">Management API</h2><h3 id="list-webhooks">List webhooks</h3><p><code>GET /api/integrations/webhooks</code></p>
<p>Returns current webhooks and available channels for an authenticated authorized user.</p>
<h3 id="create-webhook">Create webhook</h3><p><code>POST /api/integrations/webhooks</code></p>
<p>JSON body:</p>
<pre><code class="language-json">{
  &quot;name&quot;: &quot;Build Alerts&quot;,
  &quot;description&quot;: &quot;Posts CI results into general&quot;,
  &quot;channelSlug&quot;: &quot;general&quot;
}
</code></pre>
<p>Response includes:</p>
<ul>
<li>created webhook metadata</li>
<li>one-time secret token</li>
<li>full webhook URL</li>
</ul>
<h3 id="revoke-webhook">Revoke webhook</h3><p><code>DELETE /api/integrations/webhooks/{id}</code></p>
<p>Revocation prevents any future deliveries from that secret URL.</p>
<h2 id="incoming-delivery-endpoint">Incoming delivery endpoint</h2><p>External applications post to:</p>
<p><code>POST /api/hooks/{token}</code></p>
<p>JSON body:</p>
<pre><code class="language-json">{
  &quot;text&quot;: &quot;Deployment complete.&quot;
}
</code></pre>
<p>Notes:</p>
<ul>
<li><code>text</code> is required</li>
<li>text is limited to 4000 characters</li>
<li>the token URL is the secret</li>
<li>revoked or unknown tokens return <code>404</code></li>
</ul>
<h2 id="example">Example</h2><pre><code class="language-bash">curl -X POST https://your-domain/api/hooks/pichat_wh_your_secret_here \
  -H &#39;Content-Type: application/json&#39; \
  -d &#39;{&quot;text&quot;:&quot;Build server says the new image is healthy.&quot;}&#39;
</code></pre>
<h2 id="current-limits">Current limits</h2><p>This is intentionally a first slice, not a full app platform. It does not yet include:</p>
<ul>
<li>scoped API tokens</li>
<li>outgoing webhooks</li>
<li>slash commands</li>
<li>OAuth app installs</li>
<li>bot event subscriptions</li>
<li>granular bot permissions</li>
<li>message formatting or attachments</li>
</ul>
<p>Those can be added later on top of the current webhook foundation.</p>
