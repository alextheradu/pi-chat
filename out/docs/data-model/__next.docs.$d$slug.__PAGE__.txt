3:T81e,<h1 id="data-model">Data Model</h1><p>The current codebase already has a broad Prisma schema even though the UI is still catching up.</p>
<h2 id="core-identity">Core identity</h2><ul>
<li><code>User</code>: primary identity record</li>
<li><code>Role</code>: access level such as <code>ADMIN</code>, <code>MODERATOR</code>, <code>MEMBER</code>, or <code>GUEST</code></li>
<li><code>Invite</code>: invite-based exception path for users outside the configured <code>ALLOWED_DOMAIN</code></li>
</ul>
<h2 id="channels-and-messaging">Channels and messaging</h2><ul>
<li><code>Channel</code></li>
<li><code>ChannelMember</code></li>
<li><code>Message</code></li>
<li><code>Reaction</code></li>
<li><code>Attachment</code></li>
<li><code>PinnedMessage</code></li>
<li><code>LinkPreview</code></li>
</ul>
<p>These models are the backbone for team communication.</p>
<h2 id="direct-communication">Direct communication</h2><ul>
<li><code>DirectMessage</code></li>
<li><code>DMAttachment</code></li>
<li><code>GroupDM</code></li>
<li><code>GroupDMMember</code></li>
<li><code>GroupDMMessage</code></li>
</ul>
<h2 id="collaboration">Collaboration</h2><ul>
<li><code>Task</code></li>
<li><code>Subdivision</code></li>
<li><code>SubdivisionMember</code></li>
<li><code>Poll</code></li>
<li><code>PollOption</code></li>
<li><code>PollVote</code></li>
</ul>
<h2 id="operations-and-platform">Operations and platform</h2><ul>
<li><code>PushSubscription</code></li>
<li><code>AuditLog</code></li>
<li><code>IncomingWebhook</code></li>
</ul>
<h2 id="incoming-webhooks">Incoming webhooks</h2><p><code>IncomingWebhook</code> is the first integrations-focused model in the schema.</p>
<p>It stores:</p>
<ul>
<li>the webhook name</li>
<li>optional description</li>
<li>target channel</li>
<li>creator</li>
<li>dedicated bot user</li>
<li>hashed secret token</li>
<li>token preview</li>
<li>revocation state</li>
<li>last-used timestamp</li>
</ul>
<p>The bot posts messages using the normal <code>Message</code> model, so webhook traffic lands in the same canonical message table as human activity.</p>
