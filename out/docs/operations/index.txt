11:T94e,<h1 id="operations">Operations</h1><p>This document covers the current operational commands and checks.</p>
<h2 id="local-development">Local development</h2><pre><code class="language-bash">npm ci
docker compose -f docker-compose.dev.yml up -d
npm run db:migrate:deploy
npm run db:seed
npm run dev
</code></pre>
<p>Local development defaults to <code>http://localhost:3001</code> unless you override <code>DEV_PORT</code>.</p>
<h2 id="validation">Validation</h2><pre><code class="language-bash">npm run lint
npm run typecheck
npm run build
npm run test
</code></pre>
<h2 id="smoke-tests">Smoke tests</h2><p>The repo includes smoke tests for the live runtime dependencies:</p>
<pre><code class="language-bash">npm run test:db
npm run test:minio
npm run test:health
</code></pre>
<p>Notes:</p>
<ul>
<li><code>test:db</code> checks the configured PostgreSQL connection.</li>
<li><code>test:minio</code> checks the configured MinIO connection and bucket addressability.</li>
<li><code>test:health</code> checks the combined runtime health status for PostgreSQL and MinIO.</li>
<li>You can point the smoke tests at alternate targets with <code>SMOKE_*</code> overrides such as <code>SMOKE_DATABASE_URL</code> or <code>SMOKE_MINIO_ENDPOINT</code>.</li>
</ul>
<h2 id="production-docker">Production Docker</h2><pre><code class="language-bash">docker compose up --build -d
docker compose ps
docker compose logs -f app
</code></pre>
<h2 id="runtime-startup-order">Runtime startup order</h2><p>The production app container:</p>
<ol>
<li>applies migrations</li>
<li>creates MinIO buckets</li>
<li>runs idempotent bootstrap seed data</li>
<li>starts the Next.js server</li>
</ol>
<h2 id="health">Health</h2><p>Use:</p>
<pre><code class="language-bash">curl http://127.0.0.1:3000/api/health
</code></pre>
<p>Expected result:</p>
<ul>
<li><code>db: ok</code></li>
<li><code>minio: ok</code></li>
</ul>
<h2 id="useful-scripts">Useful scripts</h2><ul>
<li><code>npm run db:generate</code></li>
<li><code>npm run db:migrate:deploy</code></li>
<li><code>npm run db:seed</code></li>
<li><code>npm run db:seed:runtime</code></li>
<li><code>npm run storage:init</code></li>
</ul>
<h2 id="integration-operations">Integration operations</h2><ul>
<li>create webhook from <code>/integrations</code></li>
<li>copy the one-time webhook URL immediately</li>
<li>revoke and recreate if a secret leaks</li>
</ul>
